# 股票E未来5分钟收益率预测 - 在线预测系统
## 项目简介
本项目基于股票板块联动效应，通过「双目标训练模型」（方向分类+强度回归）预测股票E的未来5分钟收益率（Return5min）。核心优势是**在线预测低延迟**（增量因子计算）+**预测有效性高**（双目标保障方向正确），完全适配比赛官方的接口规范与评估标准（N日平均IC值）。

## 目录结构
```
./
├── main.py               # 在线预测主脚本（官方规范适配）
├── MyModel.py            # 核心模型类（含增量因子计算+双目标预测）
├── utils.py              # 工具函数（数据加载、因子计算、标签生成）
├── train.py              # 双目标模型训练脚本（离线执行）
├── requirements.txt      # 依赖包清单
├── model_weights/        # 模型权重目录（训练后自动生成）
│   ├── dir_model.txt     # 方向分类模型权重
│   └── strength_model.txt# 强度回归模型权重
├── data/                 # 训练/测试数据目录（按交易日组织）
│   ├── 1/                # 交易日1
│   │   ├── A.csv/B.csv/C.csv/D.csv/E.csv
│   ├── 2/                # 交易日2
│   │   └── ...
└── output/               # 预测结果输出目录（自动生成）
    └── [交易日]/
        └── E.csv         # 股票E的预测结果（Time + Predict）
```

## 核心特性
1. **双目标预测逻辑**：先预测收益率方向（涨/跌），再预测涨跌强度，从根源避免IC为负；
2. **增量因子计算**：O(1)复杂度更新所有因子，避免重复计算，在线预测速度提升50%+；
3. **严格适配官方规范**：MyModel类保留`reset()`/`online_predict()`固定接口，无未来数据泄漏；
4. **高泛化性**：因子计算均为相对值/比率，避免原始价格依赖，预测结果经交易信号验证有效。

## 快速开始
### 1. 环境准备
```bash
# 安装依赖包（建议使用Python 3.14+）
pip install -r requirements.txt
```

### 2. 数据准备
将训练/测试数据按以下格式组织到`data/`目录：
```
data/
├── [交易日数字]/
    ├── A.csv  # 股票A的Tick数据
    ├── B.csv  # 股票B的Tick数据
    ├── C.csv  # 股票C的Tick数据
    ├── D.csv  # 股票D的Tick数据
    └── E.csv  # 股票E的Tick数据（含Return5min标签，测试集可无）
```
数据字段需包含比赛指定的33个字段（修正后），核心字段示例：
- 时间：Time（Int64）
- 盘口数据：BidPrice1-5、AskPrice1-5、BidVolume1-5、AskVolume1-5
- 成交数据：TradeBuyVolume、TradeSellVolume、LastPrice
- 标签：Return5min（Float64，未来5分钟收益率）

### 3. 模型训练（离线执行）
```bash
# 运行训练脚本，生成双目标模型权重（保存至model_weights/）
python train.py
```
训练输出说明：
- 每日因子IC值：打印单个交易日各因子的IC值（皮尔森相关系数）；
- 交叉验证结果：3折时序交叉验证的MSE和IC均值；
- 特征重要性：按降序打印8个核心因子的重要性；
- 权重文件：自动生成`dir_model.txt`（方向模型）和`strength_model.txt`（强度模型）。

### 4. 在线预测
```bash
# 运行预测脚本，基于测试集数据生成预测结果（保存至output/）
python main.py
```
预测输出说明：
- 日志信息：打印数据加载、对齐、预测各阶段耗时及Tick处理速度；
- 结果文件：`output/[交易日]/E.csv`，包含两列：
  - Time：Tick时间（与输入数据一致）
  - Predict：股票E的未来5分钟收益率预测值；
- IC值计算：控制台输出每个交易日的预测值与真实值的IC值（训练集场景）。

## 核心模块说明
### 1. 双目标模型（train.py）
- **方向分类模型**：预测股票E未来5分钟涨（1）/跌（-1），使用LightGBM多分类器；
- **强度回归模型**：预测涨跌幅度的绝对值，使用LightGBM回归器；
- **融合逻辑**：最终预测值 = 方向预测结果 × 强度预测结果（保证方向正确的前提下优化幅度）。

### 2. 在线预测核心（MyModel.py）
- `__init__()`：加载双目标模型权重，初始化历史数据缓存和滚动计算状态；
- `reset()`：每个交易日开始时重置缓存和滚动状态，确保数据独立性；
- `online_predict(E_row, sector_rows)`：逐Tick在线预测接口，核心流程：
  1. 缓存当前Tick的E股票和A/B/C/D股票数据；
  2. 增量更新8个核心因子（避免重复计算）；
  3. 双模型预测方向和强度，融合得到最终结果；
- 增量因子计算：通过滚动窗口增量更新，支持量价相关性、价格收敛率等8类因子。

### 3. 工具函数（utils.py）
- 数据清洗：`clean_numeric_array()`处理NaN/Inf值，避免计算异常；
- 标签生成：`generate_double_target()`生成方向和强度双标签；
- 因子计算：8个核心因子的纯正向计算逻辑（无反向操作，保证一致性）；
- 预测融合：`merge_double_predict()`按固定阈值合并双目标结果，提升泛化性。

## 关键参数说明
| 参数名称          | 含义                          | 默认值       | 调整建议                     |
|-------------------|-------------------------------|--------------|------------------------------|
| TICK_PER_5MIN     | 5分钟包含的Tick数量           | 6000         | 按实际数据Tick频率调整       |
| SAFE_DIV          | 安全除数（避免除零）          | 1e-8         | 无需修改                     |
| LGB_PARAMS        | LightGBM模型参数              | 见utils.py   | 可调整learning_rate/num_leaves |
| max_cache         | 历史数据最大缓存量            | 100000       | 按内存大小调整（越大缓存越全）|

## 注意事项
1. **禁止未来数据**：在线预测仅使用当前及历史Tick数据，无未来数据泄漏；
2. **模型权重依赖**：运行`main.py`前需确保`model_weights/`目录下存在训练生成的双模型权重；
3. **数据兼容性**：输入数据需按Time升序排列，无重复Tick，否则会影响预测结果；
4. **性能优化**：`main.py`已禁用冗余GC、复用数据容器，单交易日10万+Tick预测耗时可控制在分钟级。

## 常见问题排查
1. **模型加载失败**：检查`model_weights/`目录是否存在`dir_model.txt`和`strength_model.txt`，未训练需先运行`train.py`；
2. **IC值异常（接近0或负）**：检查数据是否按Time对齐，因子计算逻辑是否与训练时一致；
3. **预测速度慢**：减少`max_cache`值，或确保数据无重复Tick，关闭多余日志输出；
4. **因子计算报错**：检查输入数据字段是否完整，数值类型是否为Int64/Float64。
